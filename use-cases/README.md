# Community Use Cases

## Share What You Built

This directory collects practical use cases for the Svg/icons Pro API, TypeScript SDK, examples, CLI workflows, and MCP-style workflows. The goal is to help developers understand real implementation patterns, not to collect links.

The SDK package is alpha until npm publication. REST examples should use live `/api/pro` behavior; `/api/v1` is not live.

## Why Share A Use Case

Good use cases help other developers:

- See how a workflow is wired end to end.
- Compare API, SDK, CLI, and MCP-style approaches.
- Reuse safe token-handling patterns.
- Learn from implementation limits and tradeoffs.
- Discover examples and recipes that map to a real problem.

## Good Use-Case Examples

- Design-system icon picker.
- Internal command palette.
- SVG sprite workflow.
- CI provenance report.
- AI coding assistant / MCP-style icon workflow.
- Framework export workflow with the CLI.

See [examples](examples/) for sample submissions. They are examples only, not customer claims.

## How To Submit

- GitHub Issue: use the use-case issue template when issues are enabled for the repository.
- Pull request: add a Markdown file under `use-cases/` using [TEMPLATE.md](TEMPLATE.md).
- GitHub Discussions: planned, only after Discussions are enabled/configured.

## What To Include

- Problem solved.
- Stack.
- API, SDK, CLI, or MCP workflow used.
- Short code snippet.
- Screenshots or GIFs if available.
- Optional public repo, demo, or project URL.
- Token-handling notes.
- Honest limitations.

## What Not To Include

- Real API tokens.
- Private customer data.
- Spam links.
- Paid or sponsored claims that are not disclosed.
- Fake claims, fake customers, or fabricated usage.
- Links unrelated to the developer workflow.

## Featuring Use Cases

Accepted submissions may be featured:

- In this repo.
- Potentially on svgicons.com after review.
- With attribution only after permission.

External links are optional. Links may be reviewed, edited, or removed. User-submitted website links may be treated as user-generated content unless editorially reviewed.

## Link Policy

Use-case submissions should explain useful developer work. They should not be link exchanges or promotional link drops. See [docs/LINK_POLICY.md](../docs/LINK_POLICY.md), [docs/COMMUNITY_USE_CASES.md](../docs/COMMUNITY_USE_CASES.md), and [docs/MODERATION.md](../docs/MODERATION.md).
